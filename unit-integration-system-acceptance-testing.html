Unit, Integration, System, Acceptance Testing

  • Unit Testing: Testing individual components or functions in isolation to ensure they work as intended.
  • Integration Testing: Testing combined parts of an application to ensure they work together correctly.
  • System Testing: Testing the complete and integrated software to verify it meets requirements.
  • Acceptance Testing: Testing conducted to determine if the system satisfies business requirements and is ready for deployment.

Purpose

Each level of testing helps catch different types of defects and ensures the software is robust and ready for release.
