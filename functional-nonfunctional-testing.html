Functional & Non-Functional Testing

Functional Testing verifies that each function of the software operates according to requirements. It includes unit, integration, system, and acceptance testing. The focus is on what the system does.

Non-Functional Testing assesses aspects like performance, security, usability, and scalability. The focus is on how the system performs under various conditions, rather than specific behaviors.

Examples

  • Functional: Login, data processing, user registration
  • Non-Functional: Load testing, security testing, compatibility testing
